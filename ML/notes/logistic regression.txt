# Logistic Regression

```
z      = weighted combination (raw score)
y_pred = σ(z) = actual prediction (probability)
```

---

Sigmoid → S-shaped (non-linear, non-convex when used in MSE)
Log     → compresses values and bends curves

---

### The function:

```
−log(σ(z))
```

is convex because:

* Sigmoid + log combination produces a function whose second derivative ≥ 0
* That’s the formal definition of convexity

Log reshapes the curve so that:

* No bumps
* No multiple minima
* Always curves upward

👉 That’s convexity

---

If MSE is used, this becomes a **non-convex graph**, so gradient descent may not work properly.

---

### Model (same as linear regression)

```
z = w1x1 + w2x2 + ... + wnxn + bias
```

---

### Loss function (applied on sigmoid(z))

```
y_pred = sigmoid(z)

L = - ( (y) * log(y_pred) + (1 - y) * log(1 - y_pred) )
      ----> convex graph having one global minimum
```

---

### Loss Intuition

#### Case 1: y = 1

```
i.   y_pred = 1   → loss = 0
ii.  y_pred = 0   → loss = inf
```

---

#### Case 2: y = 0

```
i.   y_pred = 1   → loss = inf
ii.  y_pred = 0   → loss = 0
```

---

### Optimization

Using the same gradient descent, we try to minimise this error function.

We will get the values of:

```
w1, w2, ... , wn and b
```

---

### Prediction

Now we compute:

```
z = weighted combination of inputs (using learned w and b)
```

Then apply sigmoid:

```
y_pred = 1 / (1 + e^(-z))
```

---

### Learning Behavior

Gradient descent forms the weights and bias such that:

* one class is pushed to one side
* another class is pushed to the other side

👉 forming a **linear decision boundary in feature space**

---

### Classification Rule

```
if y_pred >= 0.5:
    class = 1
else:
    class = 2
```
